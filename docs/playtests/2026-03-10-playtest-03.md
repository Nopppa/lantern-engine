# Lantern Engine Playtest 03

Date: 2026-03-10
Tester: Herra luutnantti
Build context: local v0.2.1 candidate (not yet released on GitHub)

## Raw feedback

- 3/3-tilassa peli ei käytännössä ole restartattavissa nykyisestä näkymästä.
- Legend ei saa näkyviin.
- Ilman legendiä restart ei löydy/onnistu.
- Myöskään immortality toggle ei toiminut käytännössä, koska legend/help ei ollut näkyvissä.
- Valo toimii paremmin.
- Lisähavainto: Säteen pituus alkaa aina alusta, kun se kimpoaa seinästä. Eli maksimipituutta ei oteta huomioon lähdöstä vaan vikasta kimmokkeesta.

## Interpreted findings

### Reclassified observation
- The previously suspected late-game crash now looks more like a broken or unclear end-of-run state than a confirmed hard crash.
- At 3/3, the user could not practically restart from the current state, which makes the run-complete flow misleading and functionally stuck from the player's perspective.
- The earlier range-upgrade crash report should remain unconfirmed as a distinct crash until repro is isolated more carefully.

### Confirmed improvements
- Lighting/beam readability is improved versus the prior build.

### New issues
- Legend/help visibility is broken or too unclear in practice: the tester could not bring it back on screen, which also hid discoverability for restart and immortality controls.
- End-of-run UX/state is misleading enough to be perceived as a crash because restart flow is not practically accessible from the 3/3 state.
- Beam range accounting appears wrong: after a wall bounce, beam length seems to reset from the bounce point instead of consuming a shared total range budget across the full path.

## Decision impact

Next step should focus on:
1. fix end-of-run / 3/3 state so restart is actually accessible and the game cannot appear stuck/crashed
2. fix legend/help visibility and keybinding clarity so restart + immortality are discoverable in practice
3. fix beam total-range budgeting across bounce segments

## Recommendation

Do not ship to GitHub yet. The most important currently confirmed gameplay bug is now the beam-range budgeting issue across bounces.
