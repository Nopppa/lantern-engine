# Lantern Engine Playtest 02

Date: 2026-03-10
Tester: Herra luutnantti
Build context: local v0.2.0 candidate (not yet released on GitHub)

## Raw feedback

- Beam feel: Paljon parempi, oikein hyvä lyhyt pulssi ja tuntuukin jopa aseelta.
- Prism + bounce: Nyt toimii. En päässyt testaataamaan montako pomppua mahdollista. Voisi tehdä kenttään sellaisen kapean käytävän jossa testata pomppujen lukumäärä.
- Immortality toggle: Toimii.
- Lighting / lit zones: Valaistus tulee nyt keskelle osuman saanutta vihollista. Itse laser voisi valaista myös.
- Yleisfiilis: Toimivampi, huomattavasti.
- Suurin parannus: Säteen pulssimaisuus.
- Suurin puute: Peli kaatui, kun sain viimeisimmän tason ja otin lisärangea säteelle. Lisäksi legend peittää ruutua.

## Screenshot notes

Observed from attached screenshot:
- readability legend panel overlaps the main HUD area and occupies too much on-screen space
- reward-selection state is active while the left-side information stack is visually crowded
- current lighting reads strongest as pooled circular zones; beam-path lighting itself is still underrepresented

## Interpreted findings

### Confirmed improvements
- Beam feel has crossed the line from prototype placeholder into a satisfying weapon-like pulse.
- Prism redirect + bounce continuity appears fixed in normal use.
- Immortality toggle works and removes testing friction.
- Overall build quality is clearly improved versus prior playtest.

### Remaining problems
- There is at least one reproducible or semi-reproducible crash when selecting an additional range upgrade at the latest reward stage.
- HUD/readability legend is too intrusive and should be reduced, moved, collapsible, or removed from the default gameplay view.
- Lighting pass is still incomplete relative to the intended fantasy: beam path itself should contribute more visible illumination.
- Bounce-cap testing is currently awkward; a purpose-built narrow corridor / test lane would help validate maximum bounce behavior.

## Decision

Next step should be a **stability + usability pass**, not content expansion yet.

Priority order:
1. fix crash on late range-upgrade selection
2. reduce or redesign legend/HUD obstruction
3. strengthen beam-path lighting so the beam itself illuminates space more clearly
4. optionally add a narrow bounce-test lane or deterministic debug test setup
