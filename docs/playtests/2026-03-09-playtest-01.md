# Lantern Engine Playtest 01

Date: 2026-03-09
Tester: Herra luutnantti
Build context: post-v0.1.1 / pre-v0.2.0 follow-up decisions

## Raw feedback

1. Muuten kyllä, mutta sen vaikutus kestää vaan hetken, mutta se jää kuitenkin sekunneiksi eikä ole sellainen välähdys.
2. Kyllä oli, toimi myös useamman kulman. Testausta helpottaa, jos saa itselleen toggle kuolemattomuus tässä vaiheessa.
3. Hyödyllinen varmasti joissain tilanteissa. Nyt ei tarvetta, mutta hauska. Huomio: prism noden jälkeen säde ei kimpoa seinistä.
4. Ei näin raakileessa, mutta kai tää tästä.
5. Kyllä. Valaistus isompaan osaan. Se miten se valaisee ja mitä valo tekee. Voisi tehdä taskulamppuna kuolleista kasveista eläviä tms?

## Interpreted findings

- Refraction Beam lingered too long and did not yet feel like a proper flash/pulse.
- Prism Node concept worked and supported multi-angle situations.
- Testing friction should be reduced with a toggle immortality/debug mode.
- Prism redirect broke later wall-bounce continuity.
- Current prototype did not yet create a strong "wow" moment, but the direction felt promising.
- Light should evolve from pure attack/readability into a stronger world/system interaction.
- Future fantasy direction: light as a transforming force, not only damage delivery.

## Decisions taken from this feedback

### Immediate follow-up (MVP-0.2)
- shorten beam into a real pulse/flash
- fix Prism Node so redirect does not kill later bounce logic
- add immortality toggle for testing
- increase lighting / lit-zone presence in a lightweight MVP-safe way

### Deliberately postponed
- broader "light changes the world" systems
- living plants / environmental restoration mechanics
- larger exploration or flashlight systems
- major content expansion / MVP-1 work

## Status

This playtest directly informed the MVP-0.2 implementation contract and the v0.2.0 iteration.
